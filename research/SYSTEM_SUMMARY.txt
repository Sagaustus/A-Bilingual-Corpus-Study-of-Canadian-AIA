TECHNICAL DETAILS

DATABASE ARCHITECTURE
─────────────────────

Master Table (projects):
  ├─ Fields: project_id, pdf_filename, project_title, department, 
  │          language, phase, program, annual_decisions
  ├─ Records: 1 per PDF document
  └─ Role: Links all other tables via foreign keys

Governance Table (WHERE DIVERGENCES APPEAR):
  ├─ Fields: governance_id, project_id (FK), oversight_mechanism,
  │          appeal_process, transparency_measures, 
  │          accountability_framework, external_audit
  ├─ Records: 1 per project
  └─ Role: Contains EN vs FR governance differences

Other Tables:
  ├─ systems: system_purpose, system_description, affected_population
  ├─ stakeholders: respondent_name, respondent_title, respondent_email
  ├─ risk_areas: risk_area, severity, affected_groups
  ├─ mitigations: mitigation_strategy, implementation_status
  └─ key_findings: biases_identified, fairness_issues, transparency_gaps

Foreign Key Relationships:
  - All tables reference projects.project_id
  - mitigations also references risk_areas.risk_id
  - Enforces referential integrity (PRAGMA foreign_keys = ON)

LLM EXTRACTION DETAILS
──────────────────────

Max Tokens: 2000

Extraction Prompt:

Success Rate: 100% on test documents (after prompt refinement)

Error Handling:

API Usage:
